🔄 Module 02 · The Identity Shift
Agent Brain vs. QA Brain
🚫 Agent Brain — Leave This Behind
"They were probably having a rough day."
"I've seen worse — this is fine."
"Close enough to the standard."
"I don't want to ding them on that."
"We're friends — I know they can do better."
✅ QA Brain — Build This
Does this meet the rubric? Yes or no.
What specific behaviors can I document?
What trend does this represent?
Would I score this the same way blind?
What coaching note would help them grow?
?
🔍 The Blind Review Test
"If I covered the agent's name and scored this ticket blind — would I score it the same way?"
If the answer is no, you've found your bias. Fix the score to match the rubric, not the relationship.
⚖️
The rubric is the standard
Not the friendship. Not the effort. Not the day.
🎁
Honest feedback is a gift

⚙️ Module 03 · Three Pillars of QA
Every Issue Falls into One of Three Buckets
01⚙️Process
The system or policy is broken. Multiple agents make the same mistake.

Fix: Document standards, update macros, clarify policy.
02👤People
A specific agent has a skill or knowledge gap. Policy exists — they're not following it.

Fix: Targeted coaching session.
03📦Product
The item itself causes repeat contacts. Same SKU, same failure mode across tickets.

Fix: Escalate to Product Quality team.
⚠️ Why this matters:  If you coach an agent for a broken process, you're punishing them for management's failure. Get the pillar right first — then act.

✍️ Module 05 · Calling Issues Out
The SBI Coaching Model
S
Situation — Set the context

Ticket #, date, and a one-line description. "On ticket #123456 (Snow Joe warranty, 3/20)…"

B
Behavior — Observable facts only

What specifically happened. No opinions. "The agent closed the ticket without confirming the customer's shipping address."

I
Impact — Why it matters

The consequence, plus the correct behavior. "This risks a failed shipment. Per policy, address confirmation is required before closing all replacement orders."

Three parts. Specific. Factual. Actionable.  No vague feedback. Show them exactly what to do differently.
🧠 Module 06 · Bias & Blind Spots
6 Biases That Corrupt QA Data
LENIENCY
Scores cluster 80–95% regardless of quality. Fix: Calibrate monthly.
FRIENDSHIP
Scores map to relationships, not performance. Fix: Blind reviews.
CONTEXT OVERLOAD
Factoring in the agent's rough day. Fix: Context goes in notes, never the score.
RESOLUTION TUNNEL
Resolution = everything must be fine. Fix: Read the full ticket first.
CATEGORY INFLATION
Full credit for partial behaviors. Fix: Standard not met = not full credit.
RECENCY BIAS
Last ticket colors your next score. Fix: Breaks. Random review order.
📣 Escalation — Calling It Out
You're an Intelligence Function
⚙️ Process Gap Escalation
3+ tickets showing the same inconsistency with no written policy? Document it. Name the gap specifically. Propose a fix. Take it to Ops or Team Lead.
📦 Product Signal Escalation
Same SKU, same failure mode, 5+ tickets in 30 days? Document the SKU, failure description, ticket numbers. Route to Product Quality team.
"You're the only person looking across every ticket, every agent, every brand simultaneously. If you see it and don't say it — it doesn't get fixed."
